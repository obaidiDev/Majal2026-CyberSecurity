Block 2 · What security actually is

Three properties a system
either has or doesn't.

Confidentiality

Only authorized parties can read it. The one you can never undo.

Integrity

The data is what it should be, unaltered. The quietest to break.

Availability

Authorized users can reach it. Ransomware lives here.

Every control you'll ever configure exists to produce one of these. Every attack exists to break one. When you meet a new tool, ask: which of the three does it produce?

Security isn't bolted on — it's these three properties. Availability is a security property, not IT's problem: ransomware is an availability attack and the most expensive event most orgs ever face.

Which property does each control produce?

Several chips legitimately belong in two or three buckets (TLS -> C+I, immutable backups -> I+A, log forwarding -> I+A, WORM -> I+A, separation of duties -> C+I). The check doesn't say "wrong" — it asks which of your single-bucket chips belong in more than one. That's defence-in-depth, discovered rather than announced.

Evidence first · name the mechanism

This is inference, not classification. Show only the analyst observation — the right-hand column — and make them reason backwards to the mechanism. The full row only unfolds after they commit.

The three are in tension

Every control trades against the others. MFA-on-everything: which property improves, which suffers? Let them argue two minutes. Don't resolve the MFA box — leave it unresolved in the deck as in the room.

Block 3 · How intrusions actually work

The attacker doesn't hack.
They log in.

Delete the movie image. The dominant intrusion today is valid credentials — phished, or stolen by malware from a browser — followed by ordinary admin tools used in abnormal ways. Almost nothing you investigate involves a zero-day. It'll involve PowerShell, RDP, and a service account doing something a service account shouldn't.

Your detection problem is not "find the malicious thing." The tools are legitimate, the credentials valid, every action technically authorized. Your problem is finding the abnormal use of normal things — and the only way is to know what normal looks like. Which is why today is about normal.
This is the important half: the consequence. Legitimate tools, valid creds, authorized actions. Detection = abnormal use of normal things.

The kill chain · left column vs right column

Top row = what they did. Bottom row = what lands in a log. Recon stays empty forever — no log exists. Then it flips: top hides, bottom stays. That flip is "you never see the left column" — executed, not asserted. The kill chain is a model, not a law: real intrusions skip stages, loop, run several at once.

MITRE ATT&CK · a defender's encyclopedia

Offline snapshot of T1003.001. Tactic = why (Credential Access), Technique = how (dumping), Sub-technique = specific how (LSASS), Procedure = what a named group actually did. Scroll to Detection and pause: this site tells you how to catch it. Nobody memorizes this — you look it up, every time, for your whole career.

Networking for defenders · four questions

Forget OSI today (Day 3, and every cert exam). Four questions are what you actually ask of traffic: who / which service / what / reliably-delivered. One record, four passes.

TCP vs UDP · a physics demo, not a quiz

TCP has a handshake, so a connection proves someone was really there. UDP has none, so the source can be forged — which is why floods and amplification use UDP. Flip "forge source IP": TCP stalls (SYN-ACK flies to the forged address, never returns); UDP delivers anyway.

DNS · build the tunnel, then catch it

The centrepiece — it rehearses Friday's whole premise. Grouping alone doesn't surface it; entropy alone false-positives on CDN hashes; unique-subdomain-count per parent breaks it open. Then their own name decodes back out of the log. "No individual event is suspicious. The pattern is." Unprovable by assertion, undeniable after this.

HTTP anatomy · and the odd one out

User-Agent is a string the client chooses — it can lie completely. But malware authors are lazy and leave a distinctive one. Ending the day on the least glamorous possible detection is the right note to end on.
